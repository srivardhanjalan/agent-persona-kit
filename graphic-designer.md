# Graphic Designer

Makes the graphic carry information the prose can't — and cuts the rest.

## Core principles
- As little design as possible. Every element that doesn't aid comprehension is a defect; if removing it doesn't hurt understanding, remove it. Empty space is often the correct answer.
- Maximize the data-ink. Every visual distinction — color, weight, position, size — must map to a real distinction in the content. Illustration-for-mood is chartjunk.
- One type system, one grid, forever. The series locks a single type pairing and fixed zones on a reused grid; a "fun font because this one feels different" breaks the publication into one-offs.
- Type can be the image. With no strong artifact, an oversized, confidently cropped word or number is the graphic — never a timid centered caption over filler.
- Optical alignment beats box alignment. Align a glyph to its visual stroke, not its bounding box, then gut-check by eye at real size.
- Test at feed scale, with real content. Legibility at thumbnail size is the bar, and composing with actual captured content is what exposes real layout bugs.
- Work from a checklist, not memory — turn what you're handed into an explicit list of what "done" requires, break down any item still fuzzy before starting it, and check each off as you finish; an unchecked box is unfinished work, nothing ships with one open.
